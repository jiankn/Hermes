## 为什么要构建自定义技能？

[社区技能目录](/zh/tutorials/skills/install-community-skills)覆盖了很多场景，但有时你需要针对自己工作流的定制化功能。自定义技能让你可以：

- 对接公司内部 API
- 自动化特定领域的任务
- 连接小众工具和服务
- 为团队创建可复用的工作流

## 技能架构

一个 Hermes 技能是一个自包含的模块，遵循标准接口：

```
my-skill/
├── index.ts          # 主技能文件
├── package.json      # 元数据和依赖
├── README.md         # 文档
└── test/
    └── index.test.ts # 测试
```

## 第 1 步：脚手架

```bash
hermes skill create my-weather-skill
```

```
📦 创建技能: my-weather-skill
   目录: ~/.hermes/skills/my-weather-skill/

   已创建文件:
   ├── index.ts
   ├── package.json
   └── README.md

✅ 技能脚手架已就绪！编辑 index.ts 开始开发。
```

## 第 2 步：定义技能

打开 `index.ts` 编写技能：

```typescript
import type { HermesSkill, SkillContext, SkillResult } from 'hermes-agent/skills';

const weatherSkill: HermesSkill = {
  // 元数据
  name: 'weather-lookup',
  version: '1.0.0',
  description: '查询任意城市的当前天气和预报',
  author: '你的名字',

  // 参数模式 — 告诉 Hermes 这个技能需要什么输入
  parameters: {
    city: {
      type: 'string',
      required: true,
      description: '城市名称（如 "北京"、"上海"、"Tokyo"）',
    },
    units: {
      type: 'string',
      required: false,
      default: 'metric',
      enum: ['metric', 'imperial'],
      description: '温度单位',
    },
  },

  // 所需权限
  permissions: ['network'],

  // 主执行函数
  async execute(
    params: { city: string; units?: string },
    context: SkillContext
  ): Promise<SkillResult> {
    const { city, units = 'metric' } = params;

    try {
      const apiKey = context.config.get('weather_api_key');
      if (!apiKey) {
        return {
          success: false,
          error: '天气 API Key 未配置。运行: hermes skill configure weather-lookup',
        };
      }

      // 支持中文城市名查询
      const encodedCity = encodeURIComponent(city);
      const url = `https://api.openweathermap.org/data/2.5/weather?q=${encodedCity}&units=${units}&lang=zh_cn&appid=${apiKey}`;
      const response = await fetch(url);

      if (!response.ok) {
        return {
          success: false,
          error: `城市 "${city}" 未找到。请检查拼写后重试。`,
        };
      }

      const data = await response.json();
      const tempUnit = units === 'metric' ? '°C' : '°F';

      return {
        success: true,
        output: `${data.name}（${data.sys.country}）天气:
🌡️ 温度: ${data.main.temp}${tempUnit}（体感 ${data.main.feels_like}${tempUnit}）
💧 湿度: ${data.main.humidity}%
🌤️ 天气: ${data.weather[0].description}
💨 风速: ${data.wind.speed} ${units === 'metric' ? '米/秒' : 'mph'}`,
        data: data,
      };
    } catch (error) {
      return {
        success: false,
        error: `获取天气失败: ${error.message}`,
      };
    }
  },

  // 配置向导
  async configure(context: SkillContext): Promise<void> {
    const apiKey = await context.prompt('请输入 OpenWeatherMap API Key:');
    context.config.set('weather_api_key', apiKey);
    context.log('✅ API Key 已保存！');
  },
};

export default weatherSkill;
```

> **和英文版的区别**：中文版添加了 `lang=zh_cn` 参数让天气描述返回中文，并且支持直接用中文城市名查询。

## 第 3 步：理解技能 API

### SkillContext 上下文对象

`context` 对象让你的技能访问 Hermes 的内部能力：

```typescript
interface SkillContext {
  // 持久化配置
  config: {
    get(key: string): string | undefined;
    set(key: string, value: string): void;
  };

  // 日志
  log(message: string): void;
  warn(message: string): void;
  error(message: string): void;

  // 用户交互
  prompt(message: string): Promise<string>;

  // Agent 能力
  agent: {
    ask(question: string): Promise<string>;  // 向 AI 模型提问
    memory: MemoryStore;                       // 长期记忆
    session: SessionStore;                     // 会话数据
  };

  // 文件系统（沙箱化）
  fs: {
    read(path: string): Promise<string>;
    write(path: string, content: string): Promise<void>;
    exists(path: string): Promise<boolean>;
  };
}
```

### SkillResult 返回对象

```typescript
interface SkillResult {
  success: boolean;
  output?: string;    // 展示给用户的可读文本
  data?: any;         // 结构化数据，供其他技能使用
  error?: string;     // 失败时的错误信息
}
```

### 自动化测试

创建 `test/index.test.ts`：

```typescript
import { describe, it, expect } from 'vitest';
import weatherSkill from '../index';

describe('weather-lookup', () => {
  it('正确返回有效城市的天气', async () => {
    const mockContext = createMockContext({
      config: { weather_api_key: 'test-key' },
    });

    const result = await weatherSkill.execute(
      { city: '北京' },
      mockContext
    );

    expect(result.success).toBe(true);
    expect(result.output).toContain('北京');
  });

  it('无效城市应优雅失败', async () => {
    const mockContext = createMockContext({
      config: { weather_api_key: 'test-key' },
    });

    const result = await weatherSkill.execute(
      { city: '不存在的城市12345' },
      mockContext
    );

    expect(result.success).toBe(false);
    expect(result.error).toContain('未找到');
  });

  it('未配置 API Key 时提示配置', async () => {
    const mockContext = createMockContext({ config: {} });

    const result = await weatherSkill.execute(
      { city: '上海' },
      mockContext
    );

    expect(result.success).toBe(false);
    expect(result.error).toContain('未配置');
  });
});
```

运行测试：

```bash
cd ~/.hermes/skills/my-weather-skill
npm test
```

## 最佳实践

### 错误处理

始终返回有意义的错误信息：

```typescript
// ❌ 差
return { success: false, error: '错误' };

// ✅ 好
return {
  success: false,
  error: `无法连接到 ${apiUrl}。请检查网络连接和 API Key。`,
};
```

### 优雅降级

部分功能失败时，返回部分结果：

```typescript
const weather = await getWeather(city);     // 成功
const forecast = await getForecast(city);   // 失败

return {
  success: true,
  output: `当前天气: ${weather}\n\n⚠️ 预报暂不可用（API 调用次数已达上限）`,
};
```

### 安全

- 永远不要硬编码 API Key——使用 `context.config`
- 验证所有输入
- 声明所有必要权限
- 对文件路径做防目录遍历处理
