## 启动你的第一个会话

完成 [Hermes Agent 安装](/zh/tutorials/getting-started/installation)后，就可以开始你的第一次对话了。启动 Hermes 后，你会进入一个交互式终端——就像和一个拥有超能力的同事聊天，它能访问你的文件系统、上网搜索，还掌握了一系列不断增长的技能。

在终端中启动 Hermes：

```bash
hermes
```

你会看到欢迎界面：

```
🚀 Hermes Agent 已启动就绪。
会话: default | 模型: anthropic/claude-3.5-sonnet
输入消息或使用 /help 查看命令列表。

>
```

试试输入你的第一句话：

```
> 你能帮我做什么？
```

Hermes 会回复一份能力概要——从代码生成到文件管理，从网络搜索到任务自动化。关键要理解的是，Hermes 不只*回答*问题，它还能*替你做事*。

### 响应格式说明

Hermes 的回复包含几种不同的内容块：

- **思考区块** — 内部推理过程（灰色显示，可折叠）
- **文本回复** — 自然语言回答
- **工具调用** — 代替你执行的操作（读文件、搜索网页等）
- **代码块** — 可执行代码，支持一键复制

```
> 创建一个 Python 脚本，获取当前比特币价格

🤔 思考中...
我来用 CoinGecko API 创建一个获取比特币价格的 Python 脚本...

📄 已创建: bitcoin_price.py
───────────────────────────────
import requests

def get_bitcoin_price():
    url = "https://api.coingecko.com/api/v3/simple/price"
    params = {"ids": "bitcoin", "vs_currencies": "usd,cny"}
    response = requests.get(url, params=params)
    data = response.json()
    return data["bitcoin"]

if __name__ == "__main__":
    prices = get_bitcoin_price()
    print(f"比特币价格: ${prices['usd']:,.2f} / ¥{prices['cny']:,.2f}")
───────────────────────────────

✅ 文件已保存。运行命令: python bitcoin_price.py
```

> **和英文版的区别**：中文版代码示例增加了人民币（CNY）价格显示，更符合国内用户的使用场景。

## 会话管理

会话（Session）是 Hermes 记忆的核心机制。每个会话维护独立的对话历史和上下文，让你可以同时处理多个项目而互不干扰。

### 创建新会话

```bash
hermes --session 我的项目
```

或在交互模式中切换：

```
> /session new 网页爬虫
✅ 已创建并切换到会话: 网页爬虫
```

### 查看所有会话

```
> /sessions
活跃会话:
  * 网页爬虫 (当前)
    default (3 条消息)
    我的项目 (12 条消息)
```

### 切换会话

```
> /session switch default
✅ 已切换到会话: default
```

### 会话为什么重要？

每个会话有自己的**上下文窗口**——AI 模型能"看到"的对话历史量。当会话变得很长时，较早的消息会从上下文中溢出。使用独立的会话，可以保持上下文聚焦，让回答更准确。

> **实用技巧**：每个任务或项目创建一个会话，用描述性名称：如 `博客改版`、`API调试`、`数据分析`。

## 实用对话模式

### 提问

简单问题直接问：

```
> MCP 协议是什么？

MCP（Model Context Protocol，模型上下文协议）是一个开放标准，
可以让 AI 模型连接外部工具和数据源。可以把它理解为一个万能适配器，
让 Hermes 能够对接 GitHub、数据库、各类 API 等外部服务...
```

### 下指令

明确告诉 Hermes 你想做什么：

```
> 读取 src/utils.js 文件，给 fetchData 函数加上错误处理
```

Hermes 会依次：
1. 读取文件
2. 分析 `fetchData` 函数
3. 添加 try-catch 和错误处理
4. 展示差异对比（diff）
5. 确认后再保存

### 多步骤任务

Hermes 擅长处理复杂的多步骤指令：

```
> 帮我搭建一个 Express.js API，需要以下接口：
> - GET /users - 获取用户列表
> - POST /users - 创建用户
> - GET /users/:id - 获取单个用户
> 用 TypeScript，加上参数校验，写一份 README
```

Hermes 会一次性搭建整个项目——创建多个文件、安装依赖、编写文档。

### 迭代改进

真正的威力在于迭代。Hermes 产出结果后，你可以持续优化：

```
> 加上接口限流
> 给 GET /users 加上分页
> 算了，把数据库从内存改成 SQLite
```

每次追加的指令都基于之前的上下文，Hermes 清楚你指的是哪个项目、哪些文件。

## 上下文与记忆

### 上下文如何运作

在同一个会话中，Hermes 会维护上下文。当你提到"那个文件"或"这个函数"时，它知道你指的是什么。

```
> 读一下 package.json
（Hermes 读取并显示文件内容）

> 加一个 lint 脚本进去
（Hermes 知道"进去" = package.json）

> 现在运行一下
（Hermes 知道"运行一下" = 刚加的 lint 脚本）
```

### 长期记忆

会话之外，Hermes 还有一套**跨会话持久记忆**。你可以主动存储偏好和关键信息：

```
> /memory save 我所有新项目都用 TypeScript
✅ 已存入长期记忆

> /memory save 我的部署目标固定是阿里云 ECS
✅ 已存入长期记忆
```

这样即使在新会话中，Hermes 也会默认使用 TypeScript，并优先考虑阿里云 ECS 部署方案。

> **国内常见设置**：可以把常用的信息存入记忆，比如 "使用 pnpm 而非 npm"、"代码注释用中文"、"数据库用 MySQL 而非 PostgreSQL" 等。

### 清除上下文

如果对话混乱了，想重新开始：

```
> /clear
✅ 对话历史已清除。重新开始。
```

## 常用命令速查

| 命令 | 说明 |
|------|------|
| `/help` | 显示所有可用命令 |
| `/session new <名称>` | 创建新会话 |
| `/session switch <名称>` | 切换会话 |
| `/sessions` | 列出所有会话 |
| `/clear` | 清除对话历史 |
| `/memory save <文本>` | 存入长期记忆 |
| `/memory list` | 查看已存记忆 |
| `/model <名称>` | 切换 AI 模型 |
| `/exit` | 退出 Hermes |

## 新手常见错误

### 错误 1：描述太模糊

```
❌ "修复我的代码"
✅ "修复 src/parser.ts 第 42 行的 TypeError —— split() 方法被调用在了 undefined 上"
```

### 错误 2：缺少上下文

```
❌ "部署这个"
✅ "把这个 Node.js 应用部署到我的阿里云 ECS（IP: 47.xx.xx.xx），使用 PM2 管理进程"
```

### 错误 3：忽略确认提示

Hermes 在执行破坏性操作（删除文件、覆盖代码）前会要求确认。务必仔细检查变更内容后再确认。
