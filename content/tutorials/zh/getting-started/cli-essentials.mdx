## Hermes CLI 全览

Hermes Agent 的命令行界面是你与 Agent 交互的主要方式。虽然可以[将 Hermes 连接到 Telegram](/zh/tutorials/messaging/telegram-bot-setup) 或 Discord 实现手机端使用，但 CLI 提供了最强大和灵活的体验。

[安装完成](/zh/tutorials/getting-started/installation)后，在终端输入：

```bash
hermes --help
```

```
Hermes Agent v0.8 — 自进化 AI Agent

用法:
  hermes [选项] [消息]

选项:
  -m, --model <MODEL>       指定 AI 模型
  -s, --session <NAME>      使用或创建会话
  -c, --continue             继续上一个会话
  -f, --file <PATH>         包含文件到上下文
  -p, --pipe                 从标准输入读取
  -o, --output <PATH>       将回复写入文件
  -v, --verbose              显示详细日志
  --no-confirm               跳过确认提示
  --json                     以 JSON 格式输出
  --version                  显示版本号
  --help                     显示帮助信息

子命令:
  config                     管理配置
  session                    管理会话
  skill                      管理技能
  memory                     管理长期记忆
  cron                       管理定时任务
```

## 基础用法

### 交互模式（默认）

直接输入 `hermes` 启动交互式对话：

```bash
hermes
```

进入 REPL 环境，可以实时输入消息并接收回复。

### 单次模式

发送单条消息获取回复，无需进入交互模式：

```bash
hermes "法国的首都是什么？"
```

非常适合快速提问或脚本集成。输出到标准输出，可以管道传递：

```bash
hermes "生成一个随机 UUID" | xclip          # Linux
hermes "生成一个随机 UUID" | pbcopy         # macOS
hermes "生成一个随机 UUID" | clip            # Windows (WSL2)
```

### 文件上下文模式

将文件加入对话进行分析或修改：

```bash
# 包含单个文件
hermes -f src/app.ts "给所有路由加上错误处理"

# 包含多个文件
hermes -f src/app.ts -f src/routes.ts -f package.json "审查这个项目"

# 用通配符匹配
hermes -f "src/**/*.ts" "查找潜在的内存泄漏"
```

### 管道模式

将其他命令的输出直接传给 Hermes：

```bash
# 分析日志
cat /var/log/nginx/error.log | hermes -p "总结最近一小时的错误"

# 解释代码
cat src/parser.ts | hermes -p "用通俗易懂的中文解释这段代码"

# 处理数据
curl -s https://api.example.com/data | hermes -p "把这个 JSON 转换成 CSV 表格"

# 代码审查
git diff HEAD~3 | hermes -p "审查这些改动并给出改进建议"
```

> **国内开发常用**：管道模式特别适合与 `docker logs`、`kubectl logs`、`pm2 logs` 等命令配合使用。

## 会话管理

会话是持久化的对话线程，每个会话维护独立的历史记录和上下文状态。

### 常用会话命令

```bash
# 使用命名会话启动
hermes -s 我的项目

# 继续上一个会话
hermes -c

# 列出所有会话
hermes session list

# 查看会话详情
hermes session info 我的项目

# 删除会话
hermes session delete 旧项目

# 导出会话记录
hermes session export 我的项目 > 会话记录.md
```

### 交互模式中的会话命令

```
> /session new API重构
> /session switch 我的项目
> /sessions
> /session rename API重构 API-v2
> /session delete API-v2
```

### 会话最佳实践

1. **每个项目一个会话** — 保持上下文聚焦
2. **用描述性名称** — `博客迁移` 而不是 `session-1`
3. **定期清理** — 删除已完成的会话节省磁盘空间
4. **导出重要记录** — 删除前先导出日志

## 配置管理

Hermes 使用 YAML 配置文件，位于 `~/.hermes/config.yaml`。可以直接编辑或用 CLI 操作：

```bash
# 查看所有配置
hermes config list

# 查询某项配置
hermes config get default_model

# 修改配置
hermes config set default_model anthropic/claude-3.5-sonnet

# 重置为默认值
hermes config reset

# 用编辑器打开配置文件
hermes config edit
```

### 核心配置项

```yaml
# ~/.hermes/config.yaml
api_key: sk-or-v1-your-key-here
provider: openrouter
default_model: anthropic/claude-3.5-sonnet
temperature: 0.7
max_tokens: 4096

# 行为配置
auto_confirm: false           # 自动确认文件修改
verbose: false                # 显示详细日志
theme: dark                   # 终端主题

# 会话配置
session_dir: ~/.hermes/sessions
max_session_history: 100

# 模型别名
model_aliases:
  快: openai/gpt-4o-mini
  强: anthropic/claude-3.5-sonnet
  本地: ollama/qwen2:7b
```

完整配置说明请参考我们的[配置完全指南](/zh/tutorials/getting-started/configuration-guide)。

## 技能管理

技能（Skills）是 Hermes 的能力扩展——像插件一样教会 Hermes 新本领。

```bash
# 列出已安装技能
hermes skill list

# 搜索社区技能
hermes skill search "网页抓取"

# 安装技能
hermes skill install github-pr-review

# 卸载技能
hermes skill remove github-pr-review

# 更新所有技能
hermes skill update --all
```

## 记忆管理

Hermes 的长期记忆跨会话持久化，用来存储你的偏好、项目信息和关键上下文。

```bash
# 存储记忆
hermes memory save "我喜欢用 tab 缩进而不是空格"

# 列出所有记忆
hermes memory list

# 搜索记忆
hermes memory search "部署"

# 删除指定记忆
hermes memory delete 42

# 清空所有记忆
hermes memory clear
```

## 高级技巧

### 命令链

用单次模式构建自动化工作流：

```bash
# 生成然后运行
hermes "生成一个 Python 脚本检测 8080 端口是否开放" -o check_port.py && python check_port.py

# 分析并修复
hermes -f src/broken.ts "查找并修复 bug" --no-confirm

# 批量处理
for file in src/*.ts; do
  hermes -f "$file" "给所有导出函数添加 JSDoc 注释" --no-confirm
done
```

### 输出格式化

控制输出格式以便脚本集成：

```bash
# JSON 输出
hermes --json "列举 5 个随机城市名"

# 直接写入文件
hermes -o report.md "生成项目状态报告"

# 静默模式（不显示思考过程）
hermes --quiet "东京现在几点？"
```

### 环境变量

通过环境变量设置默认值：

```bash
export HERMES_MODEL=anthropic/claude-3.5-sonnet
export HERMES_API_KEY=sk-or-v1-your-key
export HERMES_SESSION=default
export HERMES_VERBOSE=true
```

加入 `.bashrc` 或 `.zshrc` 实现永久生效。

## 调试和排错

### 详细模式

加 `-v` 查看请求/响应详细日志：

```bash
hermes -v "你好"
```

```
[DEBUG] 连接到 OpenRouter...
[DEBUG] 模型: anthropic/claude-3.5-sonnet
[DEBUG] 发送 1 条消息 (245 tokens)
[DEBUG] 收到响应，耗时 1.2s (312 tokens)
你好！有什么可以帮你的吗？
```

### 常见问题

**"Model not found" 错误：**
```bash
hermes config get provider
hermes --list-models | grep claude
```

**响应缓慢：**
```bash
# 换用更快的模型
hermes config set default_model openai/gpt-4o-mini
```

**上下文过长：**
```bash
# 新建一个干净的会话
hermes session new 重新开始
```
