## 配置概览

Hermes Agent 高度可定制。它的每个行为——从使用哪个 AI 模型到如何处理文件权限——都可以通过 YAML 配置文件、环境变量或 CLI 参数来控制。

[安装完成](/zh/tutorials/getting-started/installation)后，默认配置文件会生成在：

```
~/.hermes/config.yaml
```

你可以直接编辑这个文件，也可以用 `hermes config` 命令快速修改。

## 完整配置文件详解

以下是带完整注释的配置文件，包含所有可用选项：

```yaml
# ~/.hermes/config.yaml
# Hermes Agent 配置文件

# ═══════════════════════════════════════════
# API 与模型设置
# ═══════════════════════════════════════════

# 你的 API 密钥（支持 OpenRouter、OpenAI、Anthropic）
api_key: sk-or-v1-your-key-here

# 默认 AI 模型
default_model: anthropic/claude-3.5-sonnet

# API 提供商: openrouter | openai | anthropic | ollama | custom
provider: openrouter

# 自定义 API 地址（用于本地部署或兼容 API）
# api_base: http://localhost:11434/v1

# 模型参数
temperature: 0.7         # 0.0 = 确定性结果, 1.0 = 创意发散
max_tokens: 4096          # 最大回复长度
top_p: 1.0                # 核采样
frequency_penalty: 0.0    # 减少重复

# 模型别名（快速切换）
model_aliases:
  快: openai/gpt-4o-mini
  强: anthropic/claude-3.5-sonnet
  码: anthropic/claude-3.5-sonnet
  省: openai/gpt-4o-mini
  本地: ollama/qwen2:7b

# ═══════════════════════════════════════════
# 行为设置
# ═══════════════════════════════════════════

# 自动确认破坏性操作（写文件、删除等）
# 警告：仅在完全信任模型时设为 true
auto_confirm: false

# 显示思考/推理过程
show_thinking: true

# 显示工具调用细节
show_tool_calls: true

# 详细日志
verbose: false

# 终端配色: dark | light | none
theme: dark

# API 错误最大重试次数
max_retries: 3

# API 调用超时（秒）
timeout: 120

# ═══════════════════════════════════════════
# 会话设置
# ═══════════════════════════════════════════

# 会话数据存储路径
session_dir: ~/.hermes/sessions

# 上下文中保留的最大消息数
max_context_messages: 50

# 上下文的最大 token 数
max_context_tokens: 100000

# 自动保存会话
auto_save: true

# ═══════════════════════════════════════════
# 记忆设置
# ═══════════════════════════════════════════

# 启用长期记忆
memory_enabled: true

# 记忆存储目录
memory_dir: ~/.hermes/memory

# 上下文中包含的最大记忆条数
max_memory_items: 20

# ═══════════════════════════════════════════
# 文件系统权限
# ═══════════════════════════════════════════

# Hermes 允许读写的目录
allowed_paths:
  - "."
  - "~/projects"
  - "~/Documents"

# Hermes 绝不允许访问的目录
blocked_paths:
  - "~/.ssh"
  - "~/.gnupg"
  - "/etc"
  - "/var"

# 需要确认的文件操作类型
confirm_operations:
  - delete       # 删除
  - overwrite    # 覆盖
  - move         # 移动

# ═══════════════════════════════════════════
# 消息集成
# ═══════════════════════════════════════════

telegram:
  enabled: false
  bot_token: ""
  allowed_users: []     # Telegram 用户 ID

discord:
  enabled: false
  bot_token: ""
  allowed_channels: []
```

## 使用 Config CLI

`hermes config` 命令提供了便捷的配置管理界面：

```bash
# 查看所有配置
hermes config list

# 查询某项值
hermes config get default_model
# 输出: anthropic/claude-3.5-sonnet

# 修改配置
hermes config set default_model openai/gpt-4o

# 恢复默认值
hermes config unset temperature

# 重置所有配置
hermes config reset

# 用编辑器打开配置文件
hermes config edit
```

### 嵌套值设置

对于嵌套配置，使用点号分隔：

```bash
# 设置 Telegram 机器人 token
hermes config set telegram.bot_token "123456:ABC-DEF"

# 启用 Telegram
hermes config set telegram.enabled true

# 添加允许的路径
hermes config append allowed_paths "~/work"
```

## 环境变量

所有配置项都可以用环境变量覆盖。格式为 `HERMES_` 加上大写的配置键名：

```bash
export HERMES_API_KEY=sk-or-v1-your-key
export HERMES_DEFAULT_MODEL=anthropic/claude-3.5-sonnet
export HERMES_PROVIDER=openrouter
export HERMES_TEMPERATURE=0.5
export HERMES_VERBOSE=true
export HERMES_AUTO_CONFIRM=false
```

### 优先级顺序

配置按以下优先级解析（由高到低）：

1. **CLI 参数** — `hermes --model gpt-4o`
2. **环境变量** — `HERMES_DEFAULT_MODEL=gpt-4o`
3. **配置文件** — `~/.hermes/config.yaml`
4. **默认值** — 内置默认值

CLI 参数优先级最高，便于为特定任务临时覆盖配置：

```bash
# 仅为本次命令使用不同的模型
hermes --model openai/gpt-4o-mini "快速问题：2+2=？"

# 临时覆盖 auto_confirm 进行批量操作
hermes --no-confirm -f "src/**/*.ts" "给所有导出添加 JSDoc 注释"
```

## 配置多套方案

为不同场景创建多套配置：

```bash
# 创建"工作"方案
cp ~/.hermes/config.yaml ~/.hermes/config.work.yaml

# 创建"个人"方案
cp ~/.hermes/config.yaml ~/.hermes/config.personal.yaml
```

快速切换：

```bash
# 使用工作方案
HERMES_CONFIG=~/.hermes/config.work.yaml hermes

# 或者在 .bashrc 中创建别名
alias hermes-work="HERMES_CONFIG=~/.hermes/config.work.yaml hermes"
alias hermes-me="HERMES_CONFIG=~/.hermes/config.personal.yaml hermes"
```

> **实用场景**：工作方案使用公司 API Key + Claude 3.5，个人方案使用自己的 Key + 本地 Qwen2。

## 安全最佳实践

### 保护 API Key

永远不要把 API Key 提交到版本控制：

```bash
echo ".env" >> .gitignore
echo ".hermes/" >> .gitignore
```

用环境变量代替配置文件存储 API Key：

```bash
# 在 .bashrc 或 .zshrc 中
export HERMES_API_KEY=sk-or-v1-your-key
```

### 限制文件访问

配置 `allowed_paths` 限制 Hermes 可读写的目录范围：

```yaml
allowed_paths:
  - "~/projects"           # 仅限项目目录
  - "/tmp/hermes-scratch"  # 临时目录

blocked_paths:
  - "~/.ssh"               # SSH 密钥
  - "~/.gnupg"             # GPG 密钥
  - "~/.hermes/config.yaml" # 防止自我修改
  - "~/passwords.txt"      # 敏感文件
```

### 保持确认提示

保持 `auto_confirm: false`（默认值），让 Hermes 在以下操作前总是询问确认：
- 写入或覆盖文件
- 删除文件
- 执行 Shell 命令
- 发起外部 API 调用

## 调优模型参数

### Temperature（温度）

控制回复的随机性：

```yaml
# 确定性输出（精确、可复现）
temperature: 0.0

# 平衡模式（推荐默认值）
temperature: 0.7

# 创意模式（更多变化、不太可预测）
temperature: 1.0
```

**建议**：代码生成用 `0.0`-`0.3`，创意写作用 `0.5`-`0.8`。

### Max Tokens（最大输出长度）

```yaml
# 简短回复（快速问答）
max_tokens: 1024

# 中等回复（日常使用）
max_tokens: 4096

# 长回复（完整文件生成）
max_tokens: 8192
```
