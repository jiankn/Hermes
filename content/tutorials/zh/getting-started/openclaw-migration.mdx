## 为什么要迁移到 Hermes Agent？

OpenClaw 是最早期的 AI Agent 框架之一，许多用户从 OpenClaw 开启了 AI Agent 之旅。Hermes Agent 是 NousResearch 打造的下一代产品，在各方面都有显著提升。

迁移后你将获得：

| 功能 | OpenClaw | Hermes Agent |
|------|----------|--------------|
| 模型支持 | ~20 个 | 200+ 个（通过 OpenRouter） |
| 记忆系统 | 基础 | 持久化 + 上下文感知 |
| 技能生态 | 有限 | 100+ 社区技能 |
| 多 Agent | ❌ | ✅ 子 Agent 编排 |
| 消息集成 | 仅 Telegram | Telegram、Discord、Slack、Signal |
| 定时任务 | ❌ | ✅ 内置 Cron |
| MCP 支持 | ❌ | ✅ 完整集成 |
| 自学习 | 有限 | ✅ 技能学习闭环 |
| 活跃度 | 维护模式 | 活跃开发（74.9k ⭐） |

> **注意**：本指南假设你使用的是 OpenClaw v2.x 或更高版本。更早的版本可能有不同的文件结构。

## 迁移前准备清单

开始之前，从 OpenClaw 安装中收集以下信息：

```bash
# 找到 OpenClaw 数据目录
echo $OPENCLAW_HOME
# 通常是: ~/.openclaw

# 查看已安装的技能
openclaw skills list > 技能备份.txt

# 导出对话历史
openclaw export --all > 对话备份.json

# 备份配置
cp ~/.openclaw/config.json ~/openclaw配置备份.json
```

确保你有：
- [ ] API Key（可以在 Hermes 中复用）
- [ ] 已安装技能列表
- [ ] 导出的对话记录（如果需要保留）
- [ ] 配置备份
- [ ] 自定义脚本或自动化任务

## 第 1 步：安装 Hermes Agent

Hermes 可以和 OpenClaw 并存安装——不需要先卸载 OpenClaw：

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/install.sh | bash
```

验证安装：

```bash
hermes --version
# hermes-agent v0.8.x
```

两个工具可以同时运行互不冲突，让你在完全切换之前充分测试 Hermes。

> **Windows 用户**：需先安装 WSL2。在 PowerShell 中运行 `wsl --install`，然后在 WSL2 终端中执行上述命令。

## 第 2 步：迁移配置

### API Key

你的 OpenRouter 或 OpenAI API Key 两个工具通用：

```bash
# 从 OpenClaw 配置中获取 API Key
cat ~/.openclaw/config.json | grep api_key

# 设置到 Hermes
hermes config set api_key "你的-api-key"
```

### 模型名称映射

OpenClaw 和 Hermes 的模型命名格式略有不同：

| OpenClaw 格式 | Hermes 格式 |
|--------------|-------------|
| `claude-3-sonnet` | `anthropic/claude-3.5-sonnet` |
| `gpt-4` | `openai/gpt-4o` |
| `llama-70b` | `meta-llama/llama-3.1-70b-instruct` |

```bash
# 设置你偏好的模型
hermes config set default_model anthropic/claude-3.5-sonnet
hermes config set provider openrouter
```

### 行为配置映射

```bash
# 如果 OpenClaw 开启了 auto_execute: true
hermes config set auto_confirm true

# 如果用了 verbose: true
hermes config set verbose true

# Temperature 参数
hermes config set temperature 0.7
```

## 第 3 步：迁移对话记录

Hermes 内置了 OpenClaw 对话迁移工具：

```bash
# 迁移所有对话
hermes migrate openclaw --source ~/.openclaw/sessions

# 迁移指定会话
hermes migrate openclaw --source ~/.openclaw/sessions/项目A

# 预览模式（不实际执行）
hermes migrate openclaw --source ~/.openclaw/sessions --dry-run
```

### 手动导入

如果自动迁移不适用，可以手动操作：

```bash
# 从 OpenClaw 导出
openclaw export --session 项目A --format json > 项目A.json

# 导入到 Hermes
hermes session import 项目A.json
```

## 第 4 步：迁移技能

大多数 OpenClaw 技能都有 Hermes 对应物：

| OpenClaw 技能 | Hermes 替代方案 |
|--------------|-----------------|
| `web-search` | `web-search`（内置） |
| `file-reader` | `file-manager`（内置） |
| `code-exec` | `code-runner`（内置） |
| `github-tool` | `hermes skill install github-integration` |
| `email-sender` | `hermes skill install email-tool` |

安装对应技能：

```bash
hermes skill install github-integration
hermes skill install email-tool
hermes skill install web-scraper
```

### 自定义技能移植

如果你写过自定义 OpenClaw 技能，需要转换格式：

```javascript
// OpenClaw 技能格式（旧）
module.exports = {
  name: "my-skill",
  execute: async (args, context) => {
    return { result: "完成" };
  }
};

// Hermes 技能格式（新）
export default {
  name: "my-skill",
  version: "1.0.0",
  description: "我的自定义技能",
  parameters: {
    input: { type: "string", required: true }
  },
  execute: async ({ input }, { agent, memory }) => {
    return { success: true, output: "完成" };
  }
};
```

主要变化：
1. 使用 ES 模块（`export default`）替代 CommonJS
2. 新增 `version` 和 `description` 字段
3. 定义 `parameters` 参数模式
4. `execute` 函数接收解构的参数和更丰富的上下文对象

## 第 5 步：迁移定时任务

如果你使用了 OpenClaw 的定时功能（通常是外部 crontab），Hermes 有内置支持：

```bash
# 旧方式：外部 crontab
# 0 9 * * * openclaw run "生成日报" --session daily

# 新方式：Hermes 内置 cron
hermes cron add "生成日报" --schedule "0 9 * * *" --session 日报汇总
```

### 管理定时任务

```bash
hermes cron list
hermes cron enable 日报
hermes cron disable 日报
hermes cron delete 日报
```

## 第 6 步：验证迁移结果

按照这个清单逐项检查：

```bash
# 1. 测试连接
hermes --test-connection

# 2. 测试对话
hermes "你好！讲个笑话听听。"

# 3. 检查迁移的会话
hermes session list

# 4. 验证技能
hermes skill list

# 5. 检查记忆
hermes memory list

# 6. 检查定时任务
hermes cron list
```

## 第 7 步：退役 OpenClaw

确认 Hermes 一切正常后，可以安全移除 OpenClaw：

```bash
# 备份 OpenClaw 数据（以防万一）
tar -czf openclaw-backup.tar.gz ~/.openclaw

# 卸载 OpenClaw
npm uninstall -g openclaw

# 删除数据目录（可选）
rm -rf ~/.openclaw
```

> **建议**：备份文件至少保留一个月，以便需要时回查旧的对话记录。

## 常见问题排查

### "对话格式无法识别"

可能是从 OpenClaw v1.x 迁移。尝试：

```bash
hermes migrate openclaw --source ~/.openclaw/sessions --format v1
```

### "技能不兼容"

部分 OpenClaw 技能没有直接对应的 Hermes 版本。查看[社区技能目录](/zh/tutorials/skills/install-community-skills)或手动移植技能。

### "API Key 被拒绝"

如果之前使用的是 OpenClaw 内置的 API 代理，需要从 [OpenRouter](https://openrouter.ai) 或你的提供商获取直接 API Key。
