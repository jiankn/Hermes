## 为什么要用 Hermes 自动化？

你已经会[和 Hermes 交互对话](/zh/tutorials/getting-started/first-conversation)了。但如果 Hermes 能在你*睡觉时*替你工作呢？通过内置 Cron 系统，Hermes 可以按计划执行任务——无需人工干预。

常见自动化场景：

- 📊 **日报** — 每天早上总结日志、指标或新闻
- 🔍 **监控** — 检查服务器健康状态并告警
- 📧 **摘要** — 编制并发送邮件周报
- 🔄 **数据同步** — 从 API 拉取数据并更新数据库
- 🧹 **维护** — 清理日志、轮转备份、更新依赖

## 创建第一个定时任务

### 启用 Cron 系统

```bash
hermes config set cron.enabled true
```

### 创建简单任务

```bash
hermes cron add "发送北京今日天气摘要" \
  --schedule "0 8 * * *" \
  --name 每日天气
```

```
✅ 定时任务已创建！
  名称: 每日天气
  计划: 0 8 * * * (每天 8:00)
  命令: 发送北京今日天气摘要
  下次运行: 明天 08:00
```

### Cron 表达式语法

```
┌───────── 分钟 (0-59)
│ ┌─────── 小时 (0-23)
│ │ ┌───── 日 (1-31)
│ │ │ ┌─── 月 (1-12)
│ │ │ │ ┌─ 星期 (0-6, 周日=0)
│ │ │ │ │
* * * * *
```

| 表达式 | 含义 |
|--------|------|
| `0 9 * * *` | 每天 9:00 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 9 * * 1-5` | 工作日 9:00 |
| `0 0 1 * *` | 每月 1 号 0:00 |
| `0 */6 * * *` | 每 6 小时 |
| `0 9,18 * * *` | 每天 9:00 和 18:00 |

### 自然语言快捷方式

Hermes 也接受可读的计划描述：

```bash
hermes cron add "检查安全更新" --schedule "every 6 hours" --name 安全检查
hermes cron add "备份数据库" --schedule "daily at 3am" --name 数据库备份
hermes cron add "生成周报" --schedule "every monday at 9am" --name 周报
```

## 管理定时任务

### 查看所有任务

```bash
hermes cron list
```

```
定时任务:
  ID  名称         计划         状态     上次运行       下次运行
  1   每日天气     0 8 * * *    ✅ 活跃  今天 08:00    明天 08:00
  2   安全检查     0 */6 * * *  ✅ 活跃  今天 12:00    今天 18:00
  3   数据库备份   0 3 * * *    ⏸️ 暂停  昨天 03:00    —
  4   周报         0 9 * * 1    ✅ 活跃  周一 09:00    下周一 09:00
```

### 暂停和恢复

```bash
hermes cron pause 数据库备份
hermes cron resume 数据库备份
```

### 修改任务

```bash
hermes cron edit 每日天气 --schedule "0 7 * * *"
hermes cron edit 每日天气 --command "发送北京和上海的天气摘要"
```

### 立即执行

```bash
hermes cron run 每日天气
```

## 实战自动化方案

### 方案 1：开发者晨间摘要

```bash
hermes cron add \
  "查看我的 GitHub 通知，总结新的 Issue 和 PR，\
   列出所有 CI 失败的构建。格式化为简洁的晨间摘要。" \
  --schedule "0 8 * * 1-5" \
  --name 晨间摘要 \
  --notify telegram
```

### 方案 2：服务器健康检查

```bash
hermes cron add \
  "检查以下服务状态: \
   - 前端: https://myapp.com/health \
   - API: https://api.myapp.com/ping \
   - 数据库: 本地执行 'pg_isready' \
   如果有任何服务异常，立即告警。" \
  --schedule "*/15 * * * *" \
  --name 健康检查 \
  --notify telegram \
  --alert-only
```

`--alert-only` 标志表示只在异常时发送通知——正常时不打扰你。

### 方案 3：依赖更新监控

```bash
hermes cron add \
  "在 ~/projects/my-app 中运行 'npm outdated'，\
   告诉我哪些包有主版本更新。忽略补丁版本。" \
  --schedule "0 10 * * 1" \
  --name 依赖监控
```

### 方案 4：日志分析与轮转

```bash
hermes cron add \
  "分析 /var/log/nginx/access.log 最近 24 小时: \
   - 访问量 Top 10 页面 \
   - 所有 5xx 错误及原因 \
   - 异常流量模式 \
   然后轮转日志文件。" \
  --schedule "0 2 * * *" \
  --name 日志分析
```

### 方案 5：自动发布博客

```bash
hermes cron add \
  "检查 ~/blog/drafts/ 中 frontmatter 包含 'publish: true' 的 .md 文件。\
   对每个文件：构建页面、部署到生产环境、\
   移动文件到 ~/blog/published/。" \
  --schedule "0 6 * * *" \
  --name 自动发布
```

## 通知渠道

定时任务可以将结果发送到任何已连接的平台：

```bash
# Telegram 通知
hermes cron add "..." --notify telegram

# 飞书通知
hermes cron add "..." --notify feishu --notify-channel "oc_xxxxx"

# 钉钉通知
hermes cron add "..." --notify dingtalk --notify-channel "xxxxx"

# 企业微信通知
hermes cron add "..." --notify wecom

# 邮件通知
hermes cron add "..." --notify email --notify-to "你@example.com"

# 多渠道同时通知
hermes cron add "..." --notify telegram --notify feishu
```

> **国内用户提示**：推荐使用飞书或企业微信作为告警渠道，无需翻墙即可接收通知。

## 完整配置示例

```yaml
# ~/.hermes/cron/tasks.yaml
tasks:
  - name: 每日天气
    schedule: "0 8 * * *"
    command: "发送北京和上海的天气摘要"
    model: "openai/gpt-4o-mini"
    session: "cron-weather"
    timeout: 60
    retries: 2
    notify:
      telegram: [123456789]
    enabled: true

  - name: 健康检查
    schedule: "*/15 * * * *"
    command: "检查服务器健康..."
    model: "openai/gpt-4o-mini"
    timeout: 30
    alert_only: true
    notify:
      telegram: [123456789]
      feishu: ["oc_xxxxx"]
    enabled: true
```

### 查看日志

```bash
# 查看某个任务的日志
hermes cron logs 每日天气

# 查看最近 N 次运行
hermes cron logs 每日天气 --last 5

# 实时查看日志
hermes cron logs --follow
```

## 省钱技巧

自动化任务用便宜的模型就足够了：

```bash
hermes cron edit 每日天气 --model openai/gpt-4o-mini
```

对于简单的健康检查和日志分析，GPT-4o Mini 完全够用，成本仅为旗舰模型的 5%。
