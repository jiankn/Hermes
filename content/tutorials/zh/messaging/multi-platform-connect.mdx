## 不仅仅是 Telegram 和 Discord

除了 [Telegram](/zh/tutorials/messaging/telegram-bot-setup) 和 [Discord](/zh/tutorials/messaging/discord-integration)，许多团队和个人还依赖其他消息平台。Hermes Agent 通过可扩展的消息框架支持连接 Slack、WhatsApp 和 Signal。

各平台特点：

| 平台 | 适合场景 | 机器人支持 | 核心优势 |
|------|---------|:---------:|---------|
| Slack | 企业团队 | ✅ 原生 | 工作流、频道、集成 |
| WhatsApp | 个人/商务 | ⚠️ 通过 API | 端到端加密、全球覆盖 |
| Signal | 隐私优先 | ⚠️ 通过桥接 | 军事级加密 |

> **国内用户补充**：如果你的团队使用飞书或企业微信，Hermes 也支持通过 Webhook 或自定义适配器接入。具体方案参考本文末尾的"国内平台集成"章节。

## Slack 集成

### 为什么选择 Slack + Hermes？

Slack 是许多技术团队的沟通核心。添加 Hermes 后：

- 在 `#dev-help` 频道回答技术问题
- 在 `#pull-requests` 频道做代码审查
- 自动总结长线程
- 按需生成报告

### 第 1 步：创建 Slack 应用

1. 访问 [api.slack.com/apps](https://api.slack.com/apps)
2. 点击 **Create New App** → **From scratch**
3. 命名 "Hermes AI"，选择你的工作区
4. 进入 **OAuth & Permissions**，添加 Bot Token Scopes：
   - `chat:write` — 发送消息
   - `channels:history` — 读取频道消息
   - `channels:read` — 查看频道信息
   - `app_mentions:read` — 检测 @提及
   - `files:read` — 读取共享文件

5. 点击 **Install to Workspace**，复制 **Bot User OAuth Token**

### 第 2 步：启用事件订阅

1. 进入 **Event Subscriptions** → 开启
2. 设置 Request URL：`https://你的服务器.com/api/slack/events`
3. 订阅机器人事件：
   - `app_mention` — 被 @提及时
   - `message.channels` — 频道消息
   - `message.im` — 私信

### 第 3 步：配置 Hermes

```bash
hermes config set slack.bot_token "xoxb-你的bot-token"
hermes config set slack.signing_secret "你的signing-secret"
hermes config set slack.enabled true
```

完整配置：

```yaml
# ~/.hermes/config.yaml
slack:
  enabled: true
  bot_token: "xoxb-你的bot-token"
  signing_secret: "你的signing-secret"
  app_token: "xapp-你的app-token"

  # 行为
  respond_to_mentions: true
  respond_to_dms: true
  use_threads: true
  reaction_acknowledgment: true  # 处理时先加 🤔 反应

  # Socket Mode（更简单，不需要公网 URL）
  socket_mode: true
```

### 第 4 步：启动

```bash
hermes slack start
```

## WhatsApp 集成

### 工作原理

WhatsApp 没有像 Telegram 那样的公开 Bot API，Hermes 通过 **WhatsApp Business API** 或社区桥接工具连接。

### 方案 A：WhatsApp Business API（官方）

适合拥有 Meta 商务账号的企业：

```yaml
whatsapp:
  enabled: true
  provider: "business-api"
  phone_number_id: "123456789"
  access_token: "EAAG..."
  webhook_url: "https://你的服务器.com/api/whatsapp"
  allowed_numbers: ["+8613800138000"]
```

### 方案 B：WhatsApp Web 桥接（个人使用）

```bash
hermes skill install whatsapp-bridge
```

首次启动时，用手机扫描二维码绑定：

```yaml
whatsapp:
  enabled: true
  provider: "web-bridge"
  allowed_numbers: ["+8613800138000"]
  session_dir: "~/.hermes/whatsapp-session"
```

> **注意**：Web 桥接仅限个人使用。商业用途请使用官方 Business API。

## Signal 集成

### 隐私至上的 AI

Signal 是最安全的消息平台。Hermes 通过 signal-cli 桥接连接。

```bash
# 安装 signal-cli
wget https://github.com/AsamK/signal-cli/releases/latest/download/signal-cli.tar.gz
tar xf signal-cli.tar.gz
sudo mv signal-cli /opt/signal-cli

# 注册手机号（需要短信验证）
/opt/signal-cli/bin/signal-cli -a +8613800138000 register
/opt/signal-cli/bin/signal-cli -a +8613800138000 verify 123456

# 安装 Hermes Signal 技能
hermes skill install signal-bridge
```

配置：

```yaml
signal:
  enabled: true
  phone_number: "+8613800138000"
  signal_cli_path: "/opt/signal-cli/bin/signal-cli"
  allowed_numbers: ["+8613900139000"]
```

## 同时运行多个平台

可以一键启动所有消息集成：

```bash
# 启动所有已启用的平台
hermes messaging start --all
```

或用 PM2 分别管理：

```bash
pm2 start "hermes telegram start" --name hermes-telegram
pm2 start "hermes discord start" --name hermes-discord
pm2 start "hermes slack start" --name hermes-slack
pm2 save
```

### 统一会话管理

所有平台共享同一套会话和记忆系统：

```
Telegram 消息 → 会话: "手机端" → Hermes AI → 回复
Discord 消息  → 会话: "discord" → Hermes AI → 回复
Slack 消息    → 会话: "工作"   → Hermes AI → 回复
```

## 国内平台集成

### 飞书集成

Hermes 支持通过飞书开放平台接入：

```bash
hermes skill install feishu-adapter
```

```yaml
feishu:
  enabled: true
  app_id: "cli_xxxxx"
  app_secret: "xxxxx"
  verification_token: "xxxxx"
  event_url: "https://你的服务器.com/api/feishu"
```

### 企业微信集成

通过企业微信自建应用接入：

```bash
hermes skill install wecom-adapter
```

```yaml
wecom:
  enabled: true
  corp_id: "ww123456"
  agent_id: 1000002
  secret: "xxxxx"
  token: "xxxxx"
  encoding_aes_key: "xxxxx"
  callback_url: "https://你的服务器.com/api/wecom"
```

> **提示**：飞书和企业微信适配器都支持自建机器人和 Webhook 两种模式。小团队推荐 Webhook 模式，配置更简单。
