## 为什么要把 Hermes 接入 Telegram？

[命令行很强大](/zh/tutorials/getting-started/cli-essentials)，但你不可能一直待在终端前。把 Hermes 连接到 Telegram 后，AI 助手就揣进了你的口袋：

- **全天候可用** — 手机、平板、任何设备随时对话
- **推送通知** — 接收定时任务和监控告警
- **媒体支持** — 发送图片、文档和语音消息进行分析
- **群组模式** — 把 Hermes 加入团队频道协作
- **会话同步** — Telegram 对话与 CLI 会话互通

## 准备工作

开始之前，确保你已经：

- [安装并配置好 Hermes Agent](/zh/tutorials/getting-started/installation)
- 拥有 Telegram 账号
- [配置好 AI 模型](/zh/tutorials/getting-started/choosing-a-model)
- 有一台云服务器或常开的电脑（实现 24/7 运行）

> **国内用户注意**：Telegram 在国内需要科学上网才能使用。你的运行 Hermes 的服务器也需要能访问 Telegram API（api.telegram.org）。推荐使用海外 VPS（如 DigitalOcean、Vultr 或腾讯云海外版）。

## 第 1 步：创建 Telegram 机器人

### 从 BotFather 获取 Bot Token

1. 打开 Telegram，搜索 `@BotFather`
2. 发送 `/newbot`
3. 输入机器人的**显示名称**（如 "我的 Hermes AI"）
4. 输入机器人的**用户名**（必须以 `bot` 结尾，如 `my_hermes_ai_bot`）
5. BotFather 会返回 API Token：

```
Done! Your bot was created successfully.
Use this token to access the HTTP API:
7123456789:AAH1234567890abcDefGhIjKlMnOpQrStUvW
```

> **重要**：这个 Token 必须保密！任何人拿到 Token 都能控制你的机器人。

### 配置机器人信息

向 @BotFather 发送以下命令：

```
/setdescription
选择你的机器人 → "基于 Hermes Agent 的个人 AI 助手"

/setcommands
选择你的机器人 → 粘贴以下内容:
help - 显示帮助信息
model - 切换 AI 模型
session - 管理对话会话
clear - 清除对话历史
memory - 管理长期记忆
status - 查看当前状态
```

## 第 2 步：配置 Hermes

### 设置 Bot Token

```bash
hermes config set telegram.bot_token "7123456789:AAH1234567890abcDefGhIjKlMnOpQrStUvW"
hermes config set telegram.enabled true
```

### 限制访问权限（重要！）

默认情况下，**任何人**找到你的机器人都能和它对话（消耗你的 API 额度）。必须限制为你的 Telegram 用户 ID：

1. 获取你的用户 ID — 向 `@userinfobot` 发送 `/start`
2. 设置允许的用户：

```bash
hermes config set telegram.allowed_users "[你的用户ID]"
```

多人使用时：

```bash
hermes config set telegram.allowed_users "[用户ID1, 用户ID2]"
```

### 完整 Telegram 配置

```yaml
# ~/.hermes/config.yaml
telegram:
  enabled: true
  bot_token: "7123456789:AAH..."
  allowed_users: [123456789]

  # 可选配置
  parse_mode: "Markdown"        # 消息格式
  max_message_length: 4096      # Telegram 消息长度限制
  split_long_messages: true     # 自动拆分长消息
  send_typing_indicator: true   # 处理时显示"正在输入..."
  webhook_mode: false           # 使用轮询模式（配置更简单）

  # 文件处理
  allow_file_upload: true       # 接收文件附件
  max_file_size: 20             # 最大文件大小 (MB)
  download_dir: "~/hermes-telegram-files"
```

## 第 3 步：启动 Telegram 机器人

### 前台模式（测试用）

```bash
hermes telegram start
```

```
🤖 Telegram 机器人已启动！
 Bot: @my_hermes_ai_bot
 模式: 轮询
 允许用户: [123456789]
 模型: anthropic/claude-3.5-sonnet

等待消息...
```

### 后台模式（生产环境）

推荐使用 PM2 进程管理器：

```bash
# 安装 PM2
npm install -g pm2

# 启动 Hermes Telegram 服务
pm2 start "hermes telegram start" --name hermes-telegram
pm2 save
pm2 startup  # 开机自启
```

或者使用 systemd（Linux 服务器）：

```bash
sudo tee /etc/systemd/system/hermes-telegram.service << 'EOF'
[Unit]
Description=Hermes Agent Telegram Bot
After=network.target

[Service]
Type=simple
User=your-username
ExecStart=/usr/local/bin/hermes telegram start
Restart=always
RestartSec=10
Environment="HERMES_API_KEY=sk-or-v1-your-key"

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl enable hermes-telegram
sudo systemctl start hermes-telegram
```

> **便宜的 VPS 推荐**：DigitalOcean $4/月、Vultr $3.5/月、RackNerd $12/年。对于 Hermes 机器人，最低配置（1核 512M 内存）就足够了。

## 第 4 步：在 Telegram 中使用 Hermes

### 基础对话

直接发消息就行：

```
你: 用 Python 怎么反转链表？

Hermes: 这是一个高效的实现：

def reverse_linked_list(head):
    prev = None
    current = head
    while current:
        next_node = current.next
        current.next = prev
        prev = current
        current = next_node
    return prev

这个方法只遍历链表一次 (时间 O(n), 空间 O(1))...
```

### 机器人命令

| 命令 | 说明 |
|------|------|
| `/help` | 显示帮助 |
| `/model 快` | 切换到快速模型 |
| `/model 强` | 切换到强力模型 |
| `/session new 项目X` | 创建新会话 |
| `/session list` | 查看所有会话 |
| `/clear` | 清除对话历史 |
| `/status` | 查看当前状态 |

### 发送文件

直接发送文件给 Hermes 进行分析：

- **代码文件** — "审查这段代码并提出改进建议"
- **日志文件** — "总结这个日志中的错误"
- **文档** — "用 3 个要点概括这份文档"
- **图片** — "描述这张图片的内容"（需要多模态模型）

### 语音消息

发送语音消息，Hermes 会自动转写并回复：

```
🎤 收到语音消息 (15 秒)
📝 转写: "帮我写一个 Docker Compose 文件，
   Node.js 应用 + PostgreSQL + Redis"

好的！这是适合你技术栈的 docker-compose.yml...
```

## 第 5 步：高级功能

### 群组模式

把 Hermes 加入群聊实现团队协作：

1. 将机器人添加到 Telegram 群组
2. 设置为管理员（需要读取消息权限）
3. 配置群组设置：

```yaml
telegram:
  group_mode:
    enabled: true
    trigger_prefix: "/ai"     # 只回复 /ai 开头的消息
    allowed_groups: [-1001234567890]  # 群组 ID
```

群组中使用：

```
/ai 写一个 SQL 查询找出重复的邮箱地址
```

## 常见问题

### 机器人不回复

```bash
# 检查进程是否运行
pm2 status hermes-telegram

# 查看日志
pm2 logs hermes-telegram --lines 50

# 手动测试连接
hermes telegram test
```

### 长消息被截断

Telegram 单条消息限制 4096 字符。启用消息拆分：

```yaml
telegram:
  split_long_messages: true
```

### 国内网络连接问题

如果服务器在国内，需要配置代理：

```yaml
telegram:
  proxy:
    type: "socks5"           # 或 "http"
    host: "127.0.0.1"
    port: 1080
```
