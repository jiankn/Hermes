## 多平台面临的挑战

当你分别将 Hermes 连接到 [Telegram](/zh/tutorials/messaging/telegram-bot-setup)、[Discord](/zh/tutorials/messaging/discord-integration) 和 [Slack](/zh/tutorials/messaging/multi-platform-connect) 时，每个平台独立运行。这带来几个问题：

- **上下文割裂** — 同一个问题在不同平台可能得到不同答案
- **记忆碎片化** — 长期记忆无法跨平台同步
- **资源浪费** — 每个连接维护独立的模型实例
- **行为不一致** — 各平台配置不同

解决方案是**统一网关**——一个 Hermes 实例路由所有平台的消息，共享同一个智能层。

## 网关架构总览

```

└──────┬───────┘  └──────┬───────┘  └──────┬───────┘
       │                 │                  │
       ▼                 ▼                  ▼
┌──────────────────────────────────────────────────┐
│              消息路由层                            │
│  ┌─────────┐ ┌──────────┐ ┌──────────────────┐  │
│  │ 认证 &  │ │ 会话     │ │ 速率限制         │  │
│  │ 权限    │ │ 管理器   │ │                  │  │
│  └─────────┘ └──────────┘ └──────────────────┘  │
└──────────────────────┬───────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────┐
│              Hermes Agent 核心                    │
│  ┌──────────┐ ┌──────────┐ ┌────────────────┐   │
│  │ AI 模型  │ │ 记忆     │ │ 技能引擎       │   │

│  └──────────┘ └──────────┘ └────────────────┘   │

```

## 搭建网关

### 配置文件

启用网关模式：

```yaml
# ~/.hermes/config.yaml
gateway:
  enabled: true
  port: 8080
  mode: "unified"               # unified | isolated

  # 会话策略
  session_strategy: "per-user"  # per-user | per-platform | shared
  # per-user: 同一用户跨平台共享上下文
  # per-platform: 各平台独立会话
  # shared: 所有人共享一个会话

  # 用户身份映射
  identity_mapping:
    enabled: true
    users:
      - name: "张三"
        telegram_id: 123456789
        discord_id: "987654321098765432"
        slack_id: "U01ABCDEF"
        feishu_id: "ou_xxxxx"
      - name: "李四"
        telegram_id: 987654321
        discord_id: "123456789098765432"

  # 平台适配器
  adapters:
    telegram:
      enabled: true
      bot_token: "7123456789:AAH..."
      priority: 1               # 数字越小优先级越高

    discord:
      enabled: true
      bot_token: "MTIzNDU2Nzg5MDEy..."
      priority: 2

    slack:
      enabled: true
      bot_token: "xoxb-..."
      priority: 3

    feishu:
      enabled: true
      app_id: "cli_xxxxx"
      app_secret: "xxxxx"
      priority: 4

  # 路由规则
  routing:
    broadcast_alerts: true       # 关键告警广播到所有平台
    alert_channels:
      telegram: [123456789]
      slack: ["C01234ABCDE"]
      feishu: ["oc_xxxxx"]

    # 按内容自动路由模型
    model_routing:
      code_review: "anthropic/claude-3.5-sonnet"
      quick_answers: "openai/gpt-4o-mini"
      default: "anthropic/claude-3.5-sonnet"
```

### 启动网关

```bash
hermes gateway start
```

## 消息路由流程

当消息从任何平台到达时：

1. **适配器**接收原始消息并标准化
2. **认证层**检查用户权限
3. **身份解析器**将平台 ID 映射到统一用户
4. **会话管理器**加载正确的会话
5. **Hermes 核心**处理消息
6. **格式化器**适配目标平台的响应格式
7. **适配器**发送格式化后的回复

### 平台特定格式化

网关自动适配各平台的格式：

```
Hermes 响应（内部）:
  "这是一个 Python 函数：\n```python\ndef hello():\n    print('你好')\n```"

→ Telegram: Markdown 等宽代码块
→ Discord: Discord 风格 Markdown + 语法高亮
→ Slack: mrkdwn + ```预格式化``` 块
→ 飞书: 富文本卡片消息 + 代码块
→ WhatsApp: 纯文本 + 缩进代码
```

### 智能模型路由

网关可以根据消息内容自动选择最佳模型：

```yaml
gateway:
  smart_routing:
    enabled: true
    rules:
      - pattern: "审查|调试|修复|重构|review|debug"
        model: "anthropic/claude-3.5-sonnet"
        reason: "代码任务需要强力模型"

      - pattern: "翻译|总结|解释|translate|summarize"
        model: "openai/gpt-4o-mini"
        reason: "简单任务，节省成本"

      - pattern: "图片|截图|照片|image|screenshot"
        model: "openai/gpt-4o"
        reason: "需要多模态能力"

      - default:
        model: "openai/gpt-4o-mini"
```

## 跨平台特性

### 统一记忆

身份映射后，用户的记忆跨所有平台持久化：

```
[Telegram] 你: /memory save 我喜欢用 Python
✅ 已存入长期记忆

[飞书] 你: 帮我写个爬虫
Hermes: 根据你的偏好，这是一个 Python 爬虫...
# 记住了在 Telegram 保存的偏好！
```

### 跨平台通知

把 cron 任务的结果发送到任何平台：

```yaml
cron:
  tasks:
    - name: "每日服务器健康检查"
      schedule: "0 9 * * *"
      command: "检查服务器状态并汇报异常"
      notify:
        telegram: [123456789]           # 发到我的手机
        slack: ["C01234ABCDE"]          # 发到 #运维 频道
        feishu: ["oc_xxxxx"]            # 发到飞书群组
```

## 生产环境部署

### Docker Compose

```yaml
# docker-compose.yml
version: '3.8'
services:
  hermes-gateway:
    image: hermes-agent:latest
    command: gateway start
    restart: always
    environment:
      - HERMES_API_KEY=sk-or-v1-your-key
      - HERMES_TELEGRAM_TOKEN=7123456789:AAH...
      - HERMES_DISCORD_TOKEN=MTIzNDU2Nzg5MDEy...
      - HERMES_SLACK_TOKEN=xoxb-...
    volumes:
      - hermes-data:/root/.hermes
    ports:
      - "8080:8080"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
      interval: 30s
      timeout: 10s
      retries: 3

volumes:
  hermes-data:
```

```bash
docker compose up -d
```

## 扩展方案

### 小规模（1-10 用户）

默认配置即可：

```bash
hermes gateway start
```

### 中等规模（10-100 用户）

用 PM2 集群：

```bash
pm2 start "hermes gateway start" --name hermes-gw -i 2
```

### 大规模（100+ 用户）

分离适配器和核心：

```
[Telegram 适配器] ──→ [Redis 队列] ──→ [Hermes Worker 池]
[Discord 适配器]  ──→ [Redis 队列] ──→ [Hermes Worker 池]
[飞书 适配器]    ──→ [Redis 队列] ──→ [Hermes Worker 池]
```

```yaml
gateway:
  queue:
    enabled: true
    backend: "redis"
    redis_url: "redis://localhost:6379"
    workers: 4
```

> **国内部署建议**：如果主要使用飞书和企业微信，建议将服务器部署在国内（阿里云/腾讯云），减少 API 延迟。只有 Telegram/Discord 适配器需要海外节点。
